**原则**
1. DRY（Don't Repeat Yourself）原则：
避免重复的代码，使用函数、模块和类等封装可复用的代码片段。
提倡模块化思维，将代码分解为可维护、可测试的小块。
1. KISS（Keep It Simple, Stupid）原则：
保持代码简单易懂，避免不必要的复杂性。
尽量使用直观的命名和结构，降低代码的认知负担。
